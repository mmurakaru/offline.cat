# E2E Tests

End-to-end tests using [Playwright](https://playwright.dev/) that walk the full upload-parse-render flow for all supported file formats.

## Setup

Install Playwright browsers (one-time):

```bash
npx playwright install chromium
```

## Running tests

Run all E2E tests (starts the dev server automatically):

```bash
npm run e2e
```

Run with the Playwright UI for debugging:

```bash
npm run e2e:ui
```

Run a specific test file:

```bash
npx playwright test e2e/upload-and-parse.spec.ts
```

Run tests matching a name pattern:

```bash
npx playwright test -g "PPTX"
```

Run in headed mode (visible browser):

```bash
npx playwright test --headed
```

## Test fixtures

`e2e/fixtures/` contains small synthetic files for each format:

| File | Format | Content |
|------|--------|---------|
| `sample.xliff` | XLIFF 1.2 | 3 trans-units |
| `sample.html` | HTML | Page with 3 text paragraphs |
| `sample.docx` | DOCX | 2 paragraphs with page dimensions |
| `sample.pptx` | PPTX | 2 slides with title and body text |

DOCX and PPTX fixtures are generated by `e2e/fixtures/generate-fixtures.ts`:

```bash
npx tsx e2e/fixtures/generate-fixtures.ts
```

## Test structure

- `upload-and-parse.spec.ts` - Upload each format, verify segments appear and correct badges render
- `canvas-modes.spec.ts` - Verify the correct editor canvas renders per format (slide canvas for PPTX, document canvas for DOCX, iframe preview for HTML, segment list for XLIFF)
